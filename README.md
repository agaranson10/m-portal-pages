# m-portal-pages

## 项目简介

`m-portal-pages` 是一个用于归档和托管多个独立 HTML 页面的仓库。这里存放的页面彼此独立，不绑定特定域名或网站，适用于演示、测试、存档或临时发布静态内容。

本仓库主要面向开发者、设计师或需要快速分享静态页面的人群，提供一种轻量级的页面存储与分发方式。

## 目录结构

```
m-portal-pages/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 页面索引（可选）
└── README.md       # 本文件
```

- **pages/**：每个子文件夹或文件代表一个单独的页面，命名清晰以便识别。
- **assets/**：存放页面间共享的样式、脚本或媒体文件。
- **index.html**：提供一个简单的导航页面，便于浏览所有归档页面（非必须）。

## 页面归档说明

- 所有页面均为纯静态 HTML，不依赖后端服务。
- 页面之间互不关联，各自独立运行。
- 归档内容可能包含示例、原型、文档片段或临时页面，不保证长期可用性。
- 建议在页面内注明创建日期或版本号，便于追踪。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件。
3. 若需部署到 Web 服务器，将整个仓库内容上传至服务器根目录即可。

## 维护说明

- 欢迎提交 Pull Request 添加新的页面，但请确保内容不违反平台政策。
- 仓库维护者会不定期清理无引用或过时的页面。
- 如有疑问或建议，可提交 Issue 进行讨论。
